The linter for rules your repo actually cares about.

Lintropy lets you store project-specific lint rules in your repo as small YAML files. Use it for architecture boundaries, banned APIs, migration policy, and team conventions that generic linters do not understand.

Repo-local rules

Rules live beside the code they govern. Review them, version them, and change them with the repo.

Structural matching

Rust rules use tree-sitter queries, so you can match syntax instead of brittle text snippets.

Agent-friendly

The format is intentionally small, explicit, and easy for humans or coding agents to author.

Quickstart

Get from zero to a real rule quickly

1

Install

Homebrew on macOS/Linux:

brew tap Typiqally/lintropy
brew install lintropy

Or build from source:

cargo install --path .
2

Scaffold config

lintropy init

This writes a root lintropy.yaml plus an example rule in .lintropy/.

3

Run the linter

lintropy check .

Lintropy discovers lintropy.yaml, loads every .rule.yaml and .rules.yaml file under .lintropy/, then reports matches.

Rule Anatomy

A rule is just YAML

This is the core idea. A rule has a message, an optional fix, a language, and a query that marks matching syntax. 

severity: warning
message: "avoid .unwrap() on `{{recv}}`; use .expect(\"...\") or ?"
fix: '{{recv}}.expect("TODO: handle error")'
language: rust
query: |
  (call_expression
    function: (field_expression
      value: (_) @recv
      field: (field_identifier) @method)
    (#eq? @method "unwrap")) @match

What Lintropy is good at

  • Architecture boundaries
  • Banned APIs or macros
  • Migration and deprecation rules
  • Repo-specific review policy
  • Agent guardrails that live with the codebase

Rule discovery

Lintropy walks up from the target path until it finds lintropy.yaml. From that repo root, it loads rules from .lintropy/ and evaluates file globs relative to that root.

Editors

Schema support and query highlighting

VS Code / Cursor

Lintropy ships JSON schemas and a query-highlighting extension for query: | blocks. 

lintropy install-query-extension vscode
lintropy install-query-extension cursor

JetBrains IDEs

You can install the bundled TextMate package for embedded query highlighting and point the IDE at the checked-in schemas. 

lintropy install-textmate-bundle --dir ~/.lintropy

FAQ

Questions people actually have first

Does this need a docs build step?

No. This site is plain HTML and CSS in docs/, so GitHub Pages can publish it directly from the repository.

Can I use it outside Rust?

Yes. Query rules are available for Rust today, and regex-based match rules cover plain text files.

Where should I look next?

The repo README.md is the full project overview. The example fixture under examples/rust-demo/ is the fastest place to see realistic rules and diagnostics.
